# 萤石摄像头智能监控告警系统 — 需求文档

## 一、项目目标

在 Mac 本机运行一个 Python 程序，通过局域网 RTSP 协议接入家中萤石摄像头的实时视频流，识别家人与陌生人，并在设定时段内检测到陌生人时在程序界面输出醒目的文字告警，同时保存截图和短视频作为证据。

本项目以“家庭自用、单机运行、先做稳定可用第一版”为目标，优先保证可接入、可识别、可告警、可留证，暂不追求多机部署、云端管理和复杂 UI。

---

## 二、项目范围

### 2.1 一期范围

- 在一台 Mac 上运行
- 使用 Python 命令行程序启动
- 配置 3-5 个萤石摄像头
- 单次运行只监控 1 个摄像头
- 通过局域网 RTSP 拉流，不依赖萤石云
- 本地保存家人照片、本地保存告警证据
- 使用程序界面文字告警进行提醒

### 2.2 非目标

- 不做桌面图形界面
- 不做移动端 App
- 不做多摄像头并发监控调度
- 不做云端存储、账号体系、远程管理后台
- 不做复杂的人体行为识别，如摔倒检测、徘徊检测

## 三、前置条件

以下条件在开发前或上线前必须确认：

- 每个目标摄像头都已验证支持局域网 RTSP 拉流
- 已拿到每个摄像头可用的 RTSP 连接参数
- Mac 与摄像头处于同一局域网，网络稳定
- 运行程序时终端窗口保持可见，便于第一时间看到文字告警
- 已准备家人样本照片，每人至少 3-5 张，尽量包含不同角度和光照

如果某个摄像头无法稳定提供 RTSP 视频流，则该摄像头不纳入一期支持范围。

## 四、功能需求

### 4.1 摄像头接入

- 家中共 3-5 个萤石摄像头
- 摄像头信息写在配置文件中，至少包含名称、RTSP 地址
- 程序启动时通过命令行参数指定要监控的摄像头
- 单个进程一次只处理 1 个摄像头的视频流
- 启动后应在合理时间内完成拉流连接，若连接失败则报错退出
- 运行中若 RTSP 断流，程序应自动重连

补充规则：

- 摄像头名称必须唯一
- RTSP 地址允许带账号密码
- 自动重连默认每 10 秒尝试一次
- 连续重连失败时程序不中断，但需要持续输出日志

### 4.2 家人录入

- 家人照片放在指定目录下，每人一个子目录
- 程序启动时自动扫描该目录并建立已知人脸特征库
- 若某个人目录为空，启动时给出警告
- 若整个家人目录不存在或没有有效照片，程序应报错退出

目录示例：

```text
known_faces/
  爸爸/
    01.jpg
    02.jpg
  妈妈/
    01.jpg
  外婆/
    01.jpg
```

### 4.3 识别逻辑

一期识别逻辑采用“两阶段判断”：

1. 先检测画面中是否出现人
2. 检测到人后，再尝试做人脸检测与家人识别

事件判断规则：

- 未检测到人：不告警
- 检测到人，且识别为家人：不告警
- 检测到人，但没有检测到可识别的人脸：按陌生人处理
- 检测到人，且人脸无法匹配到家人：按陌生人处理

这样定义的原因是：如果只在“检测到陌生人脸”时才告警，会漏掉背身、侧脸、低光、口罩遮挡等场景。

为降低误报，陌生人事件不应由单帧直接触发，应满足以下条件之一：

- 连续采样的 3 帧均判断为陌生人
- 或在 2 秒内多次判断为陌生人且达到阈值

具体阈值允许配置，但应提供默认值。

### 4.4 告警时段

- 支持每个摄像头配置多个告警时段
- 支持跨天时段，例如 `22:00-07:00`
- 时段判断以 Mac 本机时区为准
- 仅在告警时段内创建告警事件
- 非告警时段允许继续进行画面分析，但不触发文字告警

补充规则：

- `start == end` 视为全天生效
- 跨天时段按“当天开始、次日结束”解释
- 至少支持分钟级精度

### 4.5 告警方式

- 检测到陌生人事件后，在程序界面输出醒目的文字告警
- 告警文字中需包含摄像头名称、事件时间、事件类型
- 同一摄像头 5 分钟内不重复触发新的文字告警
- 告警冷却按摄像头维度独立计算

事件与冷却规则：

- 满足触发条件时，创建 1 条告警事件
- 在冷却时间内再次检测到陌生人，不再创建新的文字告警事件
- 冷却时间内允许继续保存日志和证据，但不重复刷出新的告警提示

界面表现要求：

- 程序默认在命令行界面运行，不引入桌面 GUI
- 陌生人事件发生时，在终端输出明显的告警文本
- 告警文本应显著区别于普通日志，例如增加固定前缀、分隔线或大写级别标记
- 至少展示以下信息：`[告警]`、摄像头名称、时间戳、事件说明、证据保存路径

### 4.6 证据保存

- 每次陌生人告警事件都要保存截图
- 每次陌生人告警事件都要保存短视频
- 文件名中包含时间戳和摄像头名称
- 所有证据保存到本地指定目录
- 证据目录占用磁盘空间必须受限，不能无限增长

视频规则明确如下：

- 保存告警前 5 秒与告警后 5 秒的视频片段
- 程序需维护内存中的环形视频缓冲区以保存告警前片段
- 告警触发后继续录制告警后片段，再合并为一个完整短视频
- 默认输出格式为 `mp4`

截图与视频命名建议：

```text
evidence/
  大门/
    2026-04-08_23-15-02_snapshot.jpg
    2026-04-08_23-15-02_clip.mp4
```

保留策略：

- 证据目录需配置最大占用空间
- 当证据目录达到上限后，程序自动删除最旧的历史证据文件
- 删除顺序按文件创建时间从旧到新执行，直到目录占用回到上限以下
- 清理策略应同时适用于截图和短视频
- 默认按单个 `evidence_dir` 总目录进行容量控制，而不是按摄像头分别限制
- 一期默认最大占用空间可设为 20 GB，允许通过配置调整

### 4.7 运行方式

- Mac 本机运行
- 命令行启动
- 支持通过参数指定摄像头名称
- 支持 Ctrl+C 优雅退出

启动示例：

```bash
python main.py --camera 客厅
```

优雅退出要求：

- 正在写入的截图和视频应尽量完成落盘
- 程序退出前关闭视频流、释放资源
- 退出过程写入日志

### 4.8 日志与异常处理

程序至少记录以下日志：

- 启动参数
- 当前选定摄像头
- 人脸库加载结果
- RTSP 连接成功或失败
- 断流与重连情况
- 陌生人事件触发情况
- 文字告警输出结果
- 证据文件保存路径
- 程序退出原因

异常处理要求：

- 配置文件格式错误时，启动失败并明确报错
- 指定摄像头不存在时，启动失败并明确报错
- 家人人脸库为空时，启动失败并明确报错
- RTSP 连接失败时，明确报错
- 运行中网络断开时，自动进入重连流程

## 五、配置文件结构

```yaml
# config.yaml
cameras:
  - name: "大门"
    rtsp_url: "rtsp://admin:password@192.168.1.100:554/h264"
    alert_schedules:
      - start: "00:00"
        end: "06:00"
    detect_outside_schedule: true

  - name: "客厅"
    rtsp_url: "rtsp://admin:password@192.168.1.101:554/h264"
    alert_schedules:
      - start: "22:00"
        end: "07:00"
      - start: "13:00"
        end: "14:00"
    detect_outside_schedule: true

alert:
  cooldown_minutes: 5
  stranger_frames_threshold: 3
  stranger_window_seconds: 2

faces_dir: "./known_faces"
evidence_dir: "./evidence"

video:
  pre_seconds: 5
  post_seconds: 5
  output_format: "mp4"

storage:
  max_evidence_size_gb: 20

stream:
  reconnect_interval_seconds: 10
```

配置要求：

- 配置加载失败时必须直接报错，不允许静默使用默认值
- `max_evidence_size_gb` 必须大于 0

## 六、典型使用流程

1. 用户为每位家人准备照片，放入 `known_faces/<姓名>/` 目录
2. 用户在 `config.yaml` 中填写摄像头 RTSP 地址和告警时段
3. 用户运行 `python main.py --camera 客厅`
4. 程序启动后加载人脸库并连接指定摄像头
5. 程序持续分析视频流
6. 在告警时段内，如检测到陌生人事件：
   - 保存截图
   - 保存告警前后共 10 秒的短视频
   - 在程序界面输出文字告警
   - 在 5 分钟冷却期内不重复输出新的告警提示
7. 用户按 `Ctrl+C` 退出程序，程序完成资源释放并结束

## 七、验收标准

满足以下条件可视为一期需求完成：

### 7.1 基本可用

- 可通过命令行指定摄像头启动程序
- 可成功连接已验证可用的 RTSP 视频流
- 可成功加载家人照片目录
- 可在终端或日志中看到运行状态

### 7.2 识别与告警

- 家人进入画面时，不触发文字告警
- 陌生人进入画面时，在告警时段内触发文字告警
- 陌生人进入画面时，在非告警时段内不输出告警提示
- 无法识别正脸但明确有人进入画面时，仍按陌生人处理
- 同一摄像头连续出现陌生人时，5 分钟内不重复输出新的告警提示

### 7.3 证据留存

- 每次告警都能生成截图
- 每次告警都能生成对应短视频
- 证据文件名包含时间戳和摄像头名称
- 证据文件可在本地目录中找到
- 当证据目录达到容量上限后，程序能够自动清理最旧证据
- 长时间运行后，证据目录总占用不超过配置上限

### 7.4 异常场景

- RTSP 断流后程序能够自动重连
- Ctrl+C 退出时，程序能够正常释放资源

### 7.5 建议性能目标

- 单路 1080p 视频流下可持续运行
- 从陌生人事件满足触发条件到输出文字告警，目标不超过 3 秒
- RTSP 断流后恢复连接时间目标不超过 30 秒
