# Claude Code 执行说明

## 1. 当前主线

当前项目的实现目标已经收敛为 HA 一期方案，只做两条能力：

1. 夜间门内开锁告警
2. 夜间童锁守护

不在本轮实现范围内：

- 二期 AI 增强
- AppDaemon
- 旧 Python RTSP 主流程
- 通用人物识别

## 2. 开始前必须先读的文档

按以下顺序阅读：

1. `docs/需求文档-v2-HA自动化.md`
2. `docs/plans/2026-04-10-HA一期自动化实现草案.md`
3. `docs/设备与系统清单.md`
4. `docs/验收测试方案.md`
5. `docs/任务清单.md`
6. `docs/自测清单.md`
7. `PROJECT_MEMORY.md`

若文档冲突，优先级按以上顺序处理。

## 3. 实施约束

- 只在 HA 自动化与 script 范围内设计和实现
- 不擅自恢复旧 Python 方案为主线
- 不擅自扩展到二期 AI
- 不擅自新增数据库、后台、App、网页 UI
- 不擅自编造未确认的童锁实体或服务名
- 若 HA 集成缺失，必须明确记录为阻塞，不允许静默跳过
- 【新增要求】编码阶段必须采用 teams / 多 agent 协作方式推进，不允许以单一角色一路做完全部工作

## 4. 多 agent 协作要求

Claude Code 在当前项目中，编码阶段至少要拆成以下角色：

1. 方案负责人
   - 负责读取需求文档、实现草案、设备清单
   - 负责拆分任务边界
   - 负责汇总最终结果

2. HA 自动化工程师
   - 负责门内开锁告警自动化
   - 负责夜间时段判断、冷却逻辑、触发链路

3. HA Script / 通知工程师
   - 负责连续抓拍脚本
   - 负责 Telegram 文字与图片通知
   - 负责失败兜底

4. 联调 / 验证工程师
   - 负责核对实体与服务
   - 负责按自测清单验证
   - 负责记录阻塞与风险

执行要求：

- 优先使用 Claude Code 原生 teams / subagents / 多 agent 能力
- 每个角色都要有明确职责，不允许只是换个名字重复做同一件事
- 最终汇总时必须说明各角色分别做了什么
- 若当前环境确实不支持 teams / 多 agent，必须第一时间明确汇报阻塞，不允许静默退化成单 agent

## 5. 当前要完成的内容

优先完成以下事项：

1. 确认门锁事件、门状态、童锁状态、童锁服务、摄像头快照、Telegram 发送方式
2. 产出一期 HA 自动化 YAML 与 script 草案
3. 对照自测清单完成最相关验证
4. 更新任务清单状态

## 6. 交付要求

每完成一个任务，至少回填：

- 修改了哪些文件
- 当前任务状态
- 做了哪些验证
- 结果如何
- 是否存在阻塞或风险

最终交付时至少汇报：

- 已完成项
- 未完成项
- 阻塞项
- 建议优先验证的真实场景
- 参与的角色及各自产出
