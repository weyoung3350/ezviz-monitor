# Claude Code 执行说明

## 1. 角色分工

- codex 负责：
  - 输出需求文档
  - 输出任务分解与实现计划
  - 在 Claude Code 完成实现和自测后做 review 和验收测试
- Claude Code 负责：
  - 按文档实现代码
  - 自行完成最相关的自测
  - 汇总修改内容、测试结果、遗留问题

## 2. 开始执行前必须先读的文档

Claude Code 开始前，必须先完整阅读以下文档：

1. `docs/需求文档.md`
2. `docs/plans/2026-04-08-萤石摄像头告警系统实现计划.md`
3. `PROJECT_MEMORY.md`
4. `docs/自测清单.md`
5. `docs/任务清单.md`

如果以上文档之间存在冲突，优先级按以下顺序处理：

1. `docs/需求文档.md`
2. `docs/plans/2026-04-08-萤石摄像头告警系统实现计划.md`
3. `PROJECT_MEMORY.md`
4. `docs/自测清单.md`
5. `docs/任务清单.md`

## 3. 当前目标与实施约束

当前一期目标是：

- 单机运行
- 单摄像头监控
- RTSP 拉流
- 重点人物识别：`杨孝治`
- 重点摄像头：`电梯厅`
- 重点时段：夜间与清晨，以配置为准
- 触发动作：`电话告警 + 本地日志 + 证据留存`

必须遵守以下约束：

- 必须按当前需求实现，不允许继续围绕“陌生人文字告警”作为一期主目标
- 电话告警是当前一期核心能力，不允许擅自弱化为“后续再做”
- 不允许擅自引入桌面 GUI、Web 后台、数据库或云端管理系统
- 一期不考虑小米门锁相关能力
- 开发联调默认使用“电梯厅”摄像头，不要先从其他摄像头开始真机测试
- 必须优先保证：
  - 规则配置可校验
  - 告警时段判断正确
  - 杨孝治识别逻辑可验证
  - 电话告警失败路径可观测
  - 证据磁盘占用受控

## 4. 实施顺序要求

必须按实现计划中的任务顺序推进，默认顺序如下：

1. T13 配置模型切换到目标人物监护规则
2. T14 目标人物识别与多帧投票
3. T15 电话告警适配层
4. T16 监控编排切换到“杨孝治夜间外出”规则
5. T17 启动检查、自测与手工验证切换
6. T18 文档收尾与交付同步

如果某一步受阻，可以调整局部实现顺序，但不得跳过测试与校验。

## 5. 每完成一个任务后的必做动作

每完成一个任务后，Claude Code 必须：

1. 运行该任务对应的最小测试
2. 如果测试失败，继续修复直到通过
3. 简要记录本任务修改了哪些文件
4. 记录本任务执行了哪些命令
5. 说明是否存在未解决风险
6. 更新 `docs/任务清单.md` 中对应任务的状态、自测和风险信息

## 6. 自测要求

- 必须执行 `docs/自测清单.md` 中与当前实现范围对应的项目
- 不能只写“已自测通过”，必须给出：
  - 执行命令
  - 结果摘要
  - 未覆盖项
- 如果某项无法在当前环境验证，必须明确说明原因

## 7. 允许的合理简化

在不改变需求目标的前提下，允许：

- 用 mock 或假对象替代真实电话服务做自动化测试
- 先实现纯逻辑函数，再接真实 I/O
- 保留终端日志输出作为辅助观测，但不能代替电话告警
- 用 `--check` 模式做启动与配置校验
- 真机联调优先使用“电梯厅”摄像头

## 8. 不允许的行为

- 不允许擅自修改需求文档中的目标与验收标准
- 不允许把“未实现”写成“已完成”
- 不允许跳过失败测试直接堆实现
- 不允许只做 happy path，不覆盖错误处理
- 不允许把辅助画像直接写成身份硬判定
- 不允许删除或弱化证据磁盘上限控制
- 不允许把小米门锁相关能力混入当前一期实现

## 9. 遇到阻塞时的处理方式

如果遇到以下问题，Claude Code 必须中止扩展实现并明确汇报：

- 电梯厅摄像头 RTSP 无法连接
- 本机缺少关键原生依赖，导致人脸识别库无法安装
- 电话告警 SDK 或 API 在当前环境无法初始化
- 需求文档与实际环境出现明显冲突
- 需要新增超出一期范围的技术方案

汇报时至少说明：

- 阻塞点是什么
- 影响哪些任务
- 已尝试过哪些方法
- 推荐的下一步选择

## 10. 最终交付格式

Claude Code 完成实现后，最终回复必须包含以下内容：

- 已完成的任务编号
- 修改文件列表
- 新增文件列表
- 实际执行的测试命令
- 测试结果摘要
- 未完成项
- 已知风险
- 建议 codex 优先验收的 3 个场景
- `docs/任务清单.md` 已更新到最新状态

## 11. 交付完成判定

只有同时满足以下条件，才算 Claude Code 阶段完成：

- 代码已实现到当前一期范围
- 对应自测已执行并有结果
- 文档已同步更新
- 遗留问题已明确列出
- 可以进入 `docs/验收测试方案.md` 所定义的验收阶段
