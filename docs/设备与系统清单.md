#### 锁事件参数（MIoT Spec 2.1020，实测已验证）

锁事件触发时携带以下属性：

**操作方式（piid=2）：**

| 值 | 含义 | 监护意义 |
|---|------|---------|
| 2 | 指纹 | 家人正常开锁 |
| 3 | 密码 | 家人正常开锁 |
| 4 | NFC | 家人正常开锁 |
| **9** | **门内按钮开锁** | **★★★ 从室内要出门** |
| 11 | 人脸 | 家人正常开锁 |
| 12 | 掌静脉 | 家人正常开锁 |

**操作位置（piid=5）：** 1=室内，2=室外

**锁动作（piid=6）：**

| 值 | 含义 |
|---|------|
| **2** | **开锁** |
| 1 | 上锁 |
| 9 | 关门 |

#### "门内开锁"判断条件

```
event.lock_event 属性:
  操作方式 = 9  (门内按钮)
  操作位置 = 1  (室内)
  锁动作 = 2    (开锁)
```

#### 实测时间线（2026-04-09 23:05，开门→关门）

| 时间 | 事件 | 参数 |
|------|------|------|
| 23:05:28 | **锁事件** | **操作方式=9, 操作位置=1, 锁动作=2（门内开锁）** |
| 23:05:29 | 电梯厅人体 | off → on |
| 23:05:31 | 门状态 | 已上锁 → 已开锁 |
| 23:05:33 | 锁事件 | 锁动作=9（关门） |
| 23:05:34 | 门状态 | 已开锁 → 已上锁 |

> **延迟评估：MIoT 事件到 HA 延迟 ~1 秒，满足实时告警需求。**
> **门内开锁事件比门状态变化早 3 秒，是最早的外出信号。**

## 四、房屋结构与外出路径分析

### 楼层布局

```
5F  露台
    └── 萤石摄像头

4F  主卧(门窗传感器) | 次卧★杨孝治卧室(门窗传感器) | 书房 | 休闲区
    └── 主卫(人体传感器) | 次卫★杨孝治使用(人体传感器)
    └── 四楼阳台(萤石摄像头)

3F  客厅(萤石摄像头) | 厨房(人体传感器) | 电梯厅(萤石摄像头+人体传感器+门锁M30 Pro)
    └── 玄关(人体传感器) → 入户门(门锁M30 Pro) → 外出
    └── 三楼阳台(萤石摄像头)
    └── 三楼卫生间(人体传感器)
```

### 杨孝治夜间外出路径

```
[四楼次卧] → [楼梯/电梯] → [三楼电梯厅] → [玄关] → [入户门] → 外出
                                ↑              ↑           ↑
                          人体传感器 ★★★  人体传感器 ★★★  门锁事件 ★★★
                          萤石摄像头                   (操作方式=9, 锁动作=2)
```

**起夜 vs 外出的区分：**
- 起夜（四楼次卫）：次卫人体传感器触发，**不经过三楼**
- 外出：四楼次卫人体传感器不触发，**三楼电梯厅+玄关触发 + 门锁开门**

**最强单一信号：** 门锁 `lock_event` 中 `操作方式=9`(门内按钮) + `锁动作=2`(开锁) = 门内开锁，无需组合其他传感器即可确认外出意图。

**辅助确认信号：**
1. 电梯厅人体传感器触发（有人到三楼）
2. 玄关人体传感器触发（接近门口）
3. 门状态从"已上锁"→"已开锁"
