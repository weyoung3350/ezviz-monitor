# 萤石摄像头告警系统实现计划

> **给 Claude：** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**目标：** 构建一个在 Mac 本机运行的 Python 命令行程序，监控单路萤石 RTSP 视频流，识别陌生人，在命令行界面输出文字告警，并在本地按磁盘上限保存证据。

**架构：** 采用单进程、模块化架构。将配置加载、时段判断、人脸库管理、视频流处理、证据留存、文字告警输出、监控编排拆分为独立模块，优先保证每个模块可单测、可替换，再逐步接成端到端流程。

**技术栈：** Python 3.11+、`pytest`、`pydantic` 或 `dataclasses`、`PyYAML`、`opencv-python`、`face_recognition` 或同类人脸库、`ffmpeg` 或 OpenCV 视频写出、标准终端输出或简单文本 UI。

**默认联调摄像头：** 客厅，IP `192.168.77.118`，验证码 `HFUIJD`，编码 `H265`，分辨率 `2304x1296`。

```python
def enforce_evidence_quota(root: Path, max_bytes: int) -> None:
    total = get_directory_size(root)
    for file_path in list_evidence_files_by_age(root):
        if total <= max_bytes:
            break
        size = file_path.stat().st_size
        file_path.unlink(missing_ok=True)
        total -= size
```

## 执行时需要盯住的风险

- `face_recognition` 在 macOS 上可能有原生依赖安装成本
- OpenCV 与 `ffmpeg` 的 RTSP 缓冲行为可能不同
- 告警前缓冲视频如果不做严格上限，可能导致内存增长
- 命令行文字告警如果样式太弱，长期值守时可能不够醒目
- 真实场景中的误报率大概率需要上线前再调阈值
