# 萤石摄像头老人夜间外出监护告警系统实现计划

> 给 Claude Code：按本计划顺序推进，并在每完成一个任务后更新 `docs/任务清单.md`。

## 1. 当前目标

基于现有单机 RTSP 监控代码，把项目目标从“陌生人文字告警”切换为“老人夜间外出监护告警”：

- 重点人物：`杨孝治`
- 重点摄像头：`电梯厅`
- 重点时段：夜间与清晨，最终以配置为准
- 核心动作：`电话告警 + 本地日志 + 证据留存`

## 2. 设计原则

- 复用已有基础能力：RTSP 拉流、配置加载、证据保存、冷却、启动检查
- 尽量小改，不做无关重构
- 先把“目标人物规则”跑通，再考虑辅助画像和扩展规则
- 先保证“杨孝治命中时一定告警”，再优化误报和漏报
- 一期不考虑小米门锁接入，后续如需接入再单独立需求

## 3. 默认联调环境

- 默认联调摄像头：`电梯厅`
- IP：`192.168.77.117`
- 验证码：`ACNPUB`
- 编码：`H265`
- 分辨率：`2304x1296`

## 4. 建议架构调整

在保留现有模块边界的前提下，调整职责如下：

- `src/config.py`
  - 支持 `monitor_rules`
  - 支持 `profiles`
  - 支持 `phone_alert`
- `src/face_registry.py`
  - 保留已知人脸加载
  - 补充“重点人物是否存在样本”的校验
- `src/vision.py`
  - 从“陌生人聚合”转为“目标人物命中聚合”
  - 支持多帧投票和阈值判断
- `src/notifier.py`
  - 保留终端日志输出
  - 新增电话告警结果输出
- `src/monitor.py`
  - 执行“摄像头 + 时段 + 人物”规则
  - 协调电话告警、证据保存和冷却
- `src/phone_alert.py`
  - 新增电话告警适配层
  - 先做阿里云语音/电话实现，保留 mock 能力
- `main.py`
  - `--check` 模式增加电话告警配置检查

## 7. 关键风险

- `face_recognition` 的准确率可能不足以直接支撑单帧识别，必须依赖多帧投票
- 现有“陌生人聚合”代码可能有较多假设，需要谨慎替换
- 电话告警外部服务虽然已开通，但仍需处理 mock、自测和失败路径
- 电梯厅真实画面角度可能与现有样本差异较大，需要补监控视角照片
