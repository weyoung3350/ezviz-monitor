- `进行中`

## 4. 任务说明

### H00 建立多 agent 分工并确认角色边界

- 目标：
  - 明确本轮使用 teams / 多 agent 协作
  - 明确各角色职责与交付边界
  - 避免多人重复改同一块内容
- 最小验收：
  - 至少形成方案负责人、HA 自动化工程师、HA Script/编排工程师、AppDaemon 通知服务工程师、联调/验证工程师 5 个角色的分工说明
  - 若环境不支持多 agent，明确标记阻塞

### 建议角色分工

- 方案负责人：
  - 读文档
  - 拆任务
  - 汇总结果
- HA 自动化工程师：
  - 负责告警自动化
- HA Script / 编排工程师：
  - 负责抓拍与通知请求脚本
- AppDaemon 通知服务工程师：
  - 负责统一通知服务
- 联调 / 验证工程师：
  - 负责实体确认、自测与风险记录

### H01 确认 HA 真实实体与服务名

- 目标：
  - 确认门锁事件实体
  - 确认门状态实体
  - 确认摄像头快照方式
  - 确认 Telegram 实际发送方式
- 最小验收：
  - 文档中不再存在关键实体/服务未确认项，或明确标记阻塞

### H02 实现夜间门内开锁告警自动化

- 目标：
  - 识别门内按钮开锁
  - 判断夜间时段
  - 判断冷却时间
  - 调用告警脚本
- 最小验收：
  - 告警自动化结构清晰
  - 5 分钟冷却策略可表达

### H03 实现连续抓拍与通知请求编排脚本

- 目标：
  - 首条文字告警
  - 连续抓拍
  - 构造 `notify_service_request`
  - 无图则保留失败兜底
- 最小验收：
  - 抓拍次数、间隔、失败路径可配置或可修改

### H04 实现 AppDaemon NotifyService

- 目标：
  - 监听 `notify_service_request`
  - 统一处理 Telegram / 电话通道
  - 统一处理静默规则与 `force_sound`
  - 保证双通道独立容错
- 最小验收：
  - 至少完成 Telegram 通道联调
  - 电话通道未就绪时有明确日志和降级说明

### H05 联调、自测与文档回填

- 目标：
  - 对照自测清单检查
  - 修正文档中的实现细节
  - 汇总阻塞与风险
- 最小验收：
  - `docs/自测清单.md` 中最相关项已回填结果
  - 本清单状态已更新
