# Claude Code 二次返工提示词

把下面这段发给 Claude Code：

```text
请不要继续新功能开发，先处理 codex 最新 review 指出的收尾问题。

这轮不是大返工，只处理 2 个点：

1. T10 监控编排主流程
2. T12 操作文档与手工验证指南

先读这些文件：
1. docs/任务清单.md
2. docs/claude-code-执行说明.md
3. docs/自测清单.md
4. docs/验收测试方案.md

你要修复的问题如下：

一、T10
当前 face_recognition 缺失时，启动链路会抛未捕获 ImportError。
要求：
- 不要让 CLI 直接吐 traceback
- 在 main.py 启动链路里把这类依赖缺失转换成明确的错误输出和非 0 退出
- 输出要能让用户知道是缺少 face_recognition 依赖
- 补充对应测试
- 测试不能只测底层函数，要覆盖 main/启动链路的行为

二、T12
docs/manual-test.md 有一处和实现不一致。
当前文档写了“启动检查会输出每人加载了多少个人脸编码”，但 run_check 实际没有做这件事。
要求：
- 修改文档，使文档与当前实现一致
- 不要顺手扩展实现去满足这句文档，优先保持最小修改
- 同时把 setuptools/pkg_resources 的兼容性风险写入任务清单对应任务的风险/阻塞字段

执行约束：
- 不要修改需求目标
- 不要改已通过 review 的任务
- 每完成一项，更新 docs/任务清单.md
- 这次处理完成后，把 T10、T12 从“待Review”继续保持或更新为适合 codex 再 review 的状态
- 保留之前的 review 记录，不要覆盖历史结论

完成后必须回报：
1. 修改了哪些文件
2. 补了哪些测试
3. 执行了哪些命令
4. 这两个问题如何修复
5. docs/任务清单.md 是否已同步更新
```
