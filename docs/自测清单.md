# 自测清单

## 1. 目的

本清单用于约束 Claude Code 在提交实现结果前必须完成的自测范围。

原则：

- 先测纯逻辑，再测模块集成
- 先测配置和边界条件，再测主流程
- 能自动化的优先自动化
- 不能自动化的要给出最小手工验证步骤

## 2. 基础要求

- 每项自测都要记录执行命令
- 每项自测都要记录结果摘要
- 失败后必须修复再重跑
- 无法执行的项目必须写明原因

## 3. 自动化测试清单

### 3.1 项目骨架

- `pytest tests/test_smoke.py -v`
- 预期：
  - `main.py` 可导入
  - 基础项目结构可运行

### 3.2 配置加载与校验

- `pytest tests/test_config.py -v`
- 至少覆盖：
  - 正常加载配置
  - 摄像头名称重复时报错
  - `max_evidence_size_gb <= 0` 报错
  - 缺少必填字段时报错
  - 跨天时段配置可被读取

### 3.3 告警时段判断

- `pytest tests/test_scheduler.py -v`
- 至少覆盖：
  - 普通时段
  - 跨天时段
  - `start == end` 全天生效
  - 非告警时段返回 false

### 3.4 证据磁盘配额清理

- `pytest tests/test_evidence.py -v`
- 至少覆盖：
  - 达到上限后删除最旧文件
  - 未达上限时不删除
  - 同时存在图片和视频时都参与清理
  - 文件已不存在时不崩溃

### 3.5 家人人脸目录扫描

- `pytest tests/test_face_registry.py -v`
- 至少覆盖：
  - 正常扫描单个人目录
  - 空目录警告
  - 根目录不存在时报错
  - 没有有效人物目录时报错

### 3.6 视频流重连控制

- `pytest tests/test_stream.py -v`
- 至少覆盖：
  - 未到重连间隔不重试
  - 到达重连间隔后可重试

### 3.7 告警冷却策略

- `pytest tests/test_alerts.py -v`
- 至少覆盖：
  - 同摄像头冷却生效
  - 不同摄像头互不影响
  - 超过冷却时间后可再次触发

### 3.8 文字告警输出

- `pytest tests/test_notifier.py -v`
- 至少覆盖：
  - 告警文本包含 `[告警]`
  - 告警文本包含摄像头名称
  - 告警文本包含时间戳
  - 告警文本包含证据路径

### 3.9 陌生人事件聚合

- `pytest tests/test_vision.py -v`
- 至少覆盖：
  - 单次命中不触发
  - 达到阈值触发
  - 超时后旧记录会被淘汰

### 3.10 监控编排主流程

- `pytest tests/test_monitor.py -v`
- 至少覆盖：
  - 告警时段内且冷却允许时触发
  - 非告警时段不触发
  - 非陌生人事件不触发

### 3.11 启动检查模式

- `pytest tests/test_main_cli.py -v`
- 至少覆盖：
  - `--camera` 参数解析正确
  - `--check` 参数解析正确

## 4. 聚合测试

- `pytest -q`
- 预期：
  - 当前实现范围内全部自动化测试通过

如果只跑了部分测试，必须明确列出未执行的测试文件及原因。

## 5. 最小手工验证

### 5.1 启动检查

- 命令：`python main.py --camera 客厅 --check`
- 预期：
  - 能完成配置加载
  - 能找到指定摄像头配置
  - 输出启动检查结果
  - 不进入长期监控循环

### 5.2 文字告警格式

- 触发一次模拟陌生人事件
- 预期：
  - 终端能看到明显的 `[告警]`
  - 摄像头名称、时间、证据路径清晰可读
  - 与普通日志有明显区别

### 5.3 证据目录上限

- 准备一个很小的测试上限
- 连续生成多份测试证据
- 预期：
  - 最旧文件被自动删除
  - 总目录占用不超过设定上限

## 6. 必须汇报的结果格式

Claude Code 在完成自测后，必须按以下格式汇报：

### 已执行命令

- 命令 1
- 命令 2

### 结果摘要

- 通过的测试
- 失败后修复再通过的测试
- 无法执行的测试

### 未覆盖项

- 未覆盖内容 1
- 原因

## 7. 不合格的自测输出示例

以下表述视为不合格：

- “我已经测过了”
- “应该没问题”
- “基础功能可用”
- “大概率通过”

必须用命令和结果说话。
