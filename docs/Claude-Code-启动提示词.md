# Claude Code 启动提示词

请在当前项目中按以下要求工作，不要跳过任何一项。

## 1. 你的角色

你负责：

- 阅读 codex 已输出的需求、计划、执行说明、自测清单和任务清单
- 按任务顺序实现代码
- 每完成一个任务就更新任务清单
- 完成与当前实现范围对应的自测
- 在最终交付时汇总修改内容、测试结果、风险和未完成项

codex 负责：

- 提供需求文档和任务拆解
- 在你完成实现和自测后进行 review 和验收测试

## 4. 执行方式

请按 `docs/plans/2026-04-08-萤石摄像头告警系统实现计划.md` 的任务顺序推进。

每完成一个任务，必须：

1. 运行对应最小测试
2. 修复直到测试通过
3. 更新 `docs/任务清单.md`
4. 在任务清单中填写：
   - 状态
   - 更新日期
   - 修改文件
   - 自测命令
   - 自测结果
   - 风险/阻塞

任务状态只允许使用：

- `未开始`
- `进行中`
- `待Review`
- `需返工`
- `已完成`
- `已阻塞`

## 5. 自测要求

你必须执行 `docs/自测清单.md` 中与当前实现范围对应的测试。

不能只说“已测试通过”，必须给出：

- 实际执行命令
- 结果摘要
- 未覆盖项
- 无法执行项及原因

## 6. 遇到阻塞时

如果遇到以下问题，不要擅自扩需求，请直接说明：

- RTSP 无法连接
- 关键依赖无法安装
- 真实环境和需求文档冲突
- 需要引入超出一期范围的新方案

汇报时请写清楚：

- 阻塞点
- 已尝试方法
- 影响范围
- 推荐下一步

完成后请按以下结构回复：

### 已完成任务

### 风险

- 点 1

先不要直接写代码。
