# Claude Code 现场验收返工提示词

把下面这段发给 Claude Code：

```text
请不要继续新功能开发，先处理 codex 在现场验收中发现的误报问题。

背景：
- 当前项目已进入现场验收阶段
- 在“已录入家人进入客厅摄像头画面”这个用例中，系统错误触发了陌生人告警
- 告警时间：2026-04-08 23:03:04
- 新生成证据：
  - evidence/客厅/2026-04-08_23-03-04_snapshot.jpg
  - evidence/客厅/2026-04-08_23-03-04_clip.mp4

这说明当前“已录入家人进入画面时不告警”这一关键需求没有满足。

请先读这些文件：
1. docs/任务清单.md
2. docs/需求文档.md
3. docs/验收测试方案.md
4. docs/claude-code-执行说明.md
5. docs/自测清单.md

这轮只处理一个问题：
- 家人已录入，但进入画面时仍被误报为陌生人

要求：

1. 先定位根因，不要盲改
重点排查这些方向：
- 家人人脸编码加载是否成功、是否足够
- 人脸识别阈值是否过严
- 缩放后的识别准确率是否过低
- “检测到人但未识别出人脸就按陌生人处理”的逻辑是否过于激进
- 家人识别命中后，是否仍可能被后续逻辑再次记为陌生人

2. 修改时遵守约束
- 不要擅自改需求目标
- 不要移除“有人但无法识别人脸时按陌生人处理”这条规则，除非你能给出和需求文档一致的更合理实现
- 优先做最小可验证修改
- 不要顺手改无关模块

3. 必须补的验证
- 补自动化测试，至少覆盖“识别为家人时不触发陌生人事件/告警”
- 如果需要，增加更贴近真实逻辑的单元测试或集成测试
- 必须说明你如何验证这次误报场景已被修复

4. 必须更新 docs/任务清单.md
- 把相关任务状态更新为适合 codex 再次 review 的状态
- 保留历史 review 记录
- 在风险/阻塞中明确写出这次误报的根因和剩余风险

5. 完成后按这个格式回报
- 根因分析
- 修改了哪些文件
- 新增/修改了哪些测试
- 执行了哪些命令
- 为什么这次修改能避免“家人入镜被误报”
- 是否还有剩余风险
```

## 使用说明

- 这份提示词用于现场验收阶段发现“家人误报”为陌生人后的定向返工
- 范围只限识别与告警逻辑，不扩展其他功能
