# 验收测试方案

## 1. 目的

本方案用于在 Claude Code 完成实现和自测后，由 codex 进行独立验收。

验收关注点不是“是否写了代码”，而是：

- 是否符合当前需求文档
- 是否围绕“老人夜间外出监护告警”主目标实现
- 是否覆盖关键场景
- 是否存在明显行为偏差

## 2. 验收前提

在进入验收前，Claude Code 必须先交付以下内容：

- 已完成实现的代码
- 自测结果汇总
- 实际执行的测试命令
- 已知风险说明
- 需要重点关注的场景说明

如果以上内容缺失，则不进入正式验收。

## 3. 验收范围

本次只验收当前一期范围：

- 单机运行
- 单摄像头监控
- RTSP 接入
- 杨孝治样本加载
- 告警时段判断
- 杨孝治目标人物识别与多帧投票
- 电话告警
- 本地日志输出
- 证据截图与短视频保存
- 证据磁盘占用控制
- 启动检查与优雅退出

以下内容不在本次验收范围：

- 通用陌生人告警
- 多摄像头并发
- 图形界面
- 任何小米门锁相关能力

## 4. 验收环境

- 运行机器：Mac
- 运行方式：命令行
- 摄像头：已验证可访问的“电梯厅”萤石 RTSP
- 样本数据：
  - 杨孝治照片目录
  - 其他家庭成员照片目录
  - 至少 1 组夜间或模拟夜间场景

## 5. 验收用例

### 用例 1：启动检查模式可用

- 前提：
  - 配置文件存在
  - 指定摄像头配置正确
  - 电话告警配置存在
- 步骤：
  1. 执行 `python main.py --camera 电梯厅 --check`
- 预期：
  - 程序完成启动检查
  - 输出摄像头、样本目录、规则和电话告警检查结果
  - 不进入持续监控
  - 无异常退出

### 用例 2：配置错误时能明确失败

- 前提：
  - 准备一份故意错误的配置
- 步骤：
  1. 启动程序
- 预期：
  - 程序直接失败
  - 明确指出错误原因
  - 不允许静默继续运行

### 用例 3：跨天时段判断正确

- 前提：
  - 配置 `22:00-07:00`
- 步骤：
  1. 构造或模拟处于告警时段和非告警时段的判断
- 预期：
  - 23:00 触发
  - 06:30 触发
  - 12:00 不触发

### 用例 4：杨孝治在电梯厅告警时段出现时触发电话告警

- 前提：
  - 当前处于告警时段
  - 杨孝治照片目录已加载
- 步骤：
  1. 让杨孝治进入电梯厅画面
- 预期：
  - 触发电话告警
  - 终端输出明显的 `[告警]`
  - 生成截图和短视频

### 用例 5：其他家庭成员出现时不触发目标告警

- 前提：
  - 当前处于告警时段
  - 其他家庭成员照片目录已加载
- 步骤：
  1. 让其他家庭成员进入电梯厅画面
- 预期：
  - 不触发“杨孝治夜间外出”电话告警
  - 不生成新的目标告警证据

### 用例 6：杨孝治在非告警时段出现时不触发

- 前提：
  - 当前不处于告警时段
- 步骤：
  1. 让杨孝治进入电梯厅画面
- 预期：
  - 不触发电话告警
  - 可记录普通日志，但不创建告警事件

### 用例 7：冷却时间生效

- 前提：
  - 同一摄像头已触发一次告警
- 步骤：
  1. 在 5 分钟内再次让杨孝治进入电梯厅画面
- 预期：
  - 不重复拨打电话
  - 不重复创建新的目标告警事件

### 用例 8：电话告警失败时仍保留证据

- 前提：
  - 模拟电话告警失败
- 步骤：
  1. 触发一次目标事件
- 预期：
  - 明确记录电话告警失败
  - 截图和短视频仍生成

### 用例 9：证据目录空间受控

- 前提：
  - 将证据目录上限设置为较小值
- 步骤：
  1. 连续制造多次目标告警，生成多份证据
- 预期：
  - 最旧证据被自动清理
  - 总占用不超过配置上限

### 用例 10：RTSP 断流后可恢复

- 前提：
  - 程序处于运行状态
- 步骤：
  1. 人为中断 RTSP 连接
  2. 恢复网络或视频流
- 预期：
  - 程序记录断流
  - 程序进入重连流程
  - 恢复后继续工作

### 用例 11：Ctrl+C 优雅退出

- 前提：
  - 程序正在运行
- 步骤：
  1. 按 `Ctrl+C`
- 预期：
  - 程序退出
  - 资源释放
  - 日志中能看到退出说明
  - 正在生成的证据尽量完成落盘

## 6. 验收判定标准

满足以下条件，判定为验收通过：

- 核心用例全部通过
- 无阻断级错误
- 行为与需求文档一致
- 无明显未说明的需求偏移

以下情况直接判定为验收不通过：

- 仍实现成“陌生人文字告警”主逻辑
- 没有电话告警
- 把其他家庭成员误报为杨孝治外出
- 证据目录可无限增长
- Claude Code 未提供完整自测记录

## 7. 验收输出格式

验收时 codex 会按以下结构输出结论：

### 结论

- 通过 / 不通过

### 发现的问题

- 问题 1
- 问题 2

### 风险与建议

- 风险 1
- 建议 1
