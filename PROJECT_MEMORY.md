# PROJECT_MEMORY

## 项目概况

- 项目名称：萤石摄像头数据分析
- 当前阶段：开发与文档收尾完成，进入“现场验收待补”阶段
- 运行环境：Mac 本机，Python 命令行程序
- 当前一期目标：单机、单摄像头监控，支持 RTSP 拉流、重点人物识别、电话告警、证据留存

## 已确认需求

- 每次运行只监控 1 个摄像头
- 摄像头通过局域网 RTSP 接入，不依赖萤石云
- 当前核心规则：
  - 目标人物：`杨孝治`
  - 目标摄像头：`电梯厅`
  - 目标时段：夜间与清晨，以配置为准
  - 动作：电话告警 + 本地日志 + 截图 + 短视频
- 告警仅在配置时段内生效，支持跨天时段
- 同一摄像头 5 分钟内不重复触发新的电话告警
- 告警时保存截图和前后各 5 秒短视频
- 证据目录需要限制最大磁盘占用，默认建议 20 GB，超限后自动删除最旧证据
- 已知设备清单已补充到需求文档
- 开发测试默认使用“电梯厅”摄像头：`192.168.77.117` / 验证码 `ACNPUB`
- 一期不考虑小米门锁接入，后续如需接入再单独立需求

## 重点人物与辅助信息

- `杨孝治`：男性，77 周岁，行动较慢，行动不便，阿尔兹海默，当前唯一重点监护对象
- `老婆`：女性，44 周岁，身材娇小
- `儿子`：男性，16 周岁

说明：

- 性别、年龄、体型、行动特征只能作为弱特征
- 身份判定仍应以人脸识别和多帧投票为主

## 当前建议架构

- `main.py` 负责启动、参数解析、优雅退出
- `src/config.py` 负责配置加载与校验，需扩展目标人物规则和电话告警配置
- `src/scheduler.py` 负责告警时段判断
- `src/face_registry.py` 负责加载家庭成员照片并生成特征库
- `src/vision.py` 负责目标人物识别与多帧投票
- `src/stream.py` 负责 RTSP 拉流与重连
- `src/evidence.py` 负责截图、短视频、磁盘容量控制
- `src/notifier.py` 负责终端日志输出
- `src/phone_alert.py` 负责电话告警适配
- `src/monitor.py` 负责规则命中、冷却、电话告警与证据编排
