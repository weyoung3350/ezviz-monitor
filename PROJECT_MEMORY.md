# PROJECT_MEMORY

## 项目概况

- 项目名称：萤石摄像头数据分析
- 当前阶段：需求澄清完成，进入实现规划阶段
- 运行环境：Mac 本机，Python 命令行程序
- 一期目标：单机、单摄像头监控，支持 RTSP 拉流、家人识别、陌生人文字告警、证据留存

## 已确认需求

- 每次运行只监控 1 个摄像头
- 摄像头通过局域网 RTSP 接入，不依赖萤石云
- 识别策略采用“两阶段判断”：
  - 先检测是否有人
  - 再做人脸识别，家人不告警，陌生人或无人脸但有人时告警
- 告警仅在配置时段内生效，支持跨天时段
- 一期告警方式为命令行界面文字告警，不引入桌面 GUI
- 同一摄像头 5 分钟内不重复输出新的文字告警
- 告警时保存截图和前后各 5 秒短视频
- 证据目录需要限制最大磁盘占用，默认建议 20 GB，超限后自动删除最旧证据
- 已知设备清单已补充到需求文档
- 开发测试默认使用“客厅”摄像头：`192.168.77.118` / 验证码 `HFUIJD`

## 当前建议架构

- `main.py` 负责启动、参数解析、优雅退出
- `src/config.py` 负责配置加载与校验
- `src/scheduler.py` 负责告警时段判断
- `src/face_registry.py` 负责加载家人照片并生成特征库
- `src/vision.py` 负责人形检测与人脸识别决策
- `src/stream.py` 负责 RTSP 拉流与重连
- `src/evidence.py` 负责截图、短视频、磁盘容量控制
- `src/notifier.py` 负责命令行文字告警输出
- `src/monitor.py` 负责事件编排、冷却和告警主流程

## 待验证事项

- 目标萤石摄像头的 RTSP 地址与认证参数是否逐台可用
- 单路 1080p 视频流在目标 Mac 上的 CPU 与内存占用
- 人形检测与人脸识别方案的准确率和延迟是否满足一期目标
- 命令行文字告警的输出样式是否足够醒目、适合长期观察
